## 3. Hidden Cache / Silent Reuse Risks

### Risk 1: `.npy` files (CRITICAL)
`compute_iou()` in the analysis script calls `np.load(npy_path)` with no staleness check.
If script 04 is not re-run, the stale `.npy` files (built from vite prompts) are silently used.
**Mitigation:** Always run scripts in order 04 → 06 → 07 → analysis. Never skip 04.

### Risk 2: Attribution graph JSON (CRITICAL)
Script 07 loads `attribution_graph_train_n48.json` by exact filename. If script 06 is not
re-run, the stale graph (vite features, different feature set) is silently loaded and all
ablation/patching results are wrong.
**Mitigation:** Always run 06 before 07.

### Risk 3: Intervention CSVs (HIGH)
`find_bridge_features()` and `analyze_c3_patching()` read the intervention CSVs directly.
If script 07 is not re-run, the stale CSVs (vite-era feature effects) are used.
**Mitigation:** Always run 07 before analysis.

### Risk 4: UI directory accumulation (LOW)
Multiple stale UI directories may accumulate. No automatic cleanup.
After rerun: verify the analysis script uses files from the current pipeline run, not the UI.
The analysis script does NOT read from UI directories — it reads from `data/results/`.
**Mitigation:** Note the new timestamped UI dir after rerun; ignore old dir.

### Risk 5: Analysis script reads baseline CSV directly (SAFE)
`check_baseline_gate()` reads `baseline_multilingual_circuits_train.csv` which is CURRENT.
Re-running script 02 is optional but harmless. The gate will pass.
